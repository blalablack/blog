html规范

发表于 2017-10-23

对于html5新标签，属性的规范总结

标签语义化

标签语义化就是让每一个标签出现在符合它意义的地方。
优点：

  • 1、条理清晰，利于阅读维护，提高团队开发效率。
  • 2、利于网站seo。
  • 3、对于阅读障碍者更加友好。
  • 4、在页面丢失样式时，保持一定的页面结构。

以上包括了基本的标签(还是要经常查表)，以后的要严格要求自己让标签语义化成为一种习惯。

脚本加载

html5新属性async
<script src="demo_async.js" async></script>用来加载外部脚本，脚本可用立即异步加载(不会阻塞页面)。
那么对于支持async的现代浏览器就可以放在head中，更具有条理性

所有浏览器，建议是放在body的下部。

HTML只关注内容

  • 除非img是页面内容，对于那些用来修饰样式的,一律用background解决
  • 不要引入一些特定的 HTML 结构来解决一些视觉设计问题

对于这种，完全可以用伪类实现

结构、表现、行为三者分离

尽量在文档和模板中只包含结构性的 HTML；

  • 1、而将所有表现代码，移入样式表中；
  • 2、将所有动作行为，移入脚本之中。
  • 3、在文档和模板中也尽量少地引入样式和脚本文件。

alt标签不为空

img标签的 alt 属性指定了替代文本，用于在图像无法显示或者用户禁用图像显示时，代替图像显示在浏览器中的内容。
假设由于下列原因用户无法查看图像，alt 属性可以为图像提供替代的信息：

  • 网速太慢
  • src 属性中的错误
  • 浏览器禁用图像
  • 用户使用的是屏幕阅读器

从SEO角度考虑，浏览器的爬虫爬不到图片的内容，所以我们要有文字告诉爬虫图片的内容。
