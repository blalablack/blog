探究this

发表于 2017-07-16 | 分类于 Javascript

此篇结合是我在看《你不知道的JavaScript》对this绑定的一些心得体会和记录。

this永远指向最后调用它的那个对象。

判读this指向的真理

函数调用方式

  • 1、作为函数直接调用
  • 2、作为方法调用
  • 3、使用函数的方法调用（call,apply,bind）
  • 4、构造函数new调用

下面我会结合书中介绍的this绑定规则（其实就是函数的几种调用方式）来做详细的介绍

this的绑定规则

一、默认绑定（作为函数直接调用）

独立函数调用，函数直接使用不带任何修饰的函数引用进行调用。

dafaultFn函数直接调用，this指向window。

二、隐式绑定（作为方法调用）

必须在对象内部包含一个指向函数的属性，并通过这个属性间接引用函数，从而把this间接（隐式）绑定到这个对象上。

在obj对象中包含一个unTypeFn的引用，在通过obj调用unTypeFn时，this被自然的绑定到obj上，没毛病。

无论怎么调用，最后调用foo的是obj1，所以是3。

1、隐式丢失

this绑定的最常见问题就是隐式绑定的函数会丢失绑定对象，也就是会应用默认绑定，从而把this绑定在全局对象上（非严格模式）。

1、值传递

obj.foo并没有直接调用，而是把它的引用地址传递给了c，所以c实际上引用的是unTypeFn函数本身，这里是默认绑定规则this指向全局。

2、参数传递

参数传递其实就是一种隐式赋值，不是直接调用obj.foo，而是把它当做值来传递。

3、回调函数

这两个都是回调函数的例子感觉跟上面的两个还是一回事，都不是立即调用，都是当做值来传递，这样会发生隐式邦定丢失。

三、显式绑定（）

在某个对象上直接强制调用函数，使用js内部的call和apply方法。

call和apply功能是一样的，只是有参数的区别。
call和apply方法是立即执行的，那么在多次调用这个方法是就可能改变this绑定的对象，于是就出现了显示绑定的变种——硬绑定的方法。

1、硬绑定（使用函数的方法调用）

ES5提供里内置的bind方法，返回一个新的绑定你指定this的函数，不会立即执行。new1.bind(obj).bind(obj1)是没啥用的他只会执行第一个。

四、new绑定（构造函数new调用）

也算是能够改变this的方式，使用new 来调用函数时，会构造一个新的对象并把它绑定到函数调用的this上，从而初始化这个新对象。

这里使用new 调用foo,会创建一个新对象bar，并把它绑定到foo调用时的this上。

优先级

new绑定>显示绑定(apply,call,bind)>隐式绑定（对象.）>默认绑定（直接调用）
